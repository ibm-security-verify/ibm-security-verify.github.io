Class WalletService

  • All Implemented Interfaces:
    com.ibm.security.verifysdk.dc.WalletServiceDescriptor 
    
public final class WalletService
 implements WalletServiceDescriptor

    The WalletService class provides implementation for the WalletServiceDescriptor interface. It offers various operations related to wallet management, including refreshing the access token, processing invitations, managing credentials, and handling proof requests.

    This service interacts with the wallet API to perform actions such as fetching credentials, processing proofs, and updating the status of credentials and verifications.

    Key Operations:

    • Refresh access tokens using the provided refresh token.

    • Preview and process invitations.

    • Process proof requests such as generating, sharing, or rejecting proofs.

    • Retrieve, update, and delete credentials.

    • Delete agents associated with the wallet.

    Error Handling: The service uses Result to wrap the responses of operations and will throw appropriate exceptions in case of errors, such as network failures or invalid API responses.

    3.0.7

         Result<TokenInfo> refreshToken(String refreshToken, String accountName, String pushToken)

        Refreshes the access token using the provided refresh token and optional metadata.

        This function communicates with the configured OAuth provider to request a new access token. It optionally includes the pushToken and accountName as additional parameters in the request, if provided. SSL verification can be bypassed based on the ignoreSsl configuration.

        Parameters:
        refreshToken - The refresh token to be used for obtaining a new access token.
        accountName - Optional name of the account to associate with the token request.
        pushToken - Optional push token to include for push notification support.
        Returns:

        A Result wrapping a TokenInfo object on success, or an error on failure.

      • processProofRequest

         VerificationInfo processProofRequest(VerificationPreviewInfo verificationPreviewInfo, VerificationAction action)

        Processes a proof request based on the given action and verification preview information.

        This function handles three main actions related to a proof request:

        • VerificationAction.GENERATE: Triggers the generation of a proof by processing the invitation URL, then updates the verification state to PROOF_GENERATED.

        • VerificationAction.SHARE: Updates the verification state to PROOF_SHARED, indicating the proof has been sent.

        • VerificationAction.REJECT: Updates the verification state to FAILED, indicating the user has declined to share the proof.

        Parameters:
        verificationPreviewInfo - The metadata and preview details of the proof request.
        action - The action to perform on the proof request: generate, share, or reject.
        Returns:

        A VerificationInfo object representing the updated state of the verification.

        Since:

        3.0.7

      • processCredential

         CredentialDescriptor processCredential(CredentialPreviewInfo credentialPreviewInfo, CredentialAction action)

        Processes a credential offer by handling the associated invitation and applying the specified action.

        This function first processes the credential invitation URL (distinct from a standard connection invitation) to move the offer into an outbound_offer state. After that, it applies the given action—either accepting or rejecting the credential—by delegating to processCredential with the credential’s identifier.

        Parameters:
        credentialPreviewInfo - The preview information for the credential offer, including its ID and invitation URL.
        action - The action to apply to the credential—either CredentialAction.ACCEPTED or CredentialAction.REJECTED.
        Returns:

        The updated CredentialDescriptor representing the credential after processing.

        Since:

        3.0.7
