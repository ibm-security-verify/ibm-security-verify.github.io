Package com.ibm.security.verifysdk.authentication.api

Class OAuthProvider

  • All Implemented Interfaces:
    
public final class OAuthProvider
extends BaseApi

    The OAuthProvider enables third-party applications to obtain limited access to an HTTP service, either on behalf of a resource owner by orchestrating an approval interaction between the resource owner and the HTTP service, or by allowing the third-party application to obtain access on its own behalf.

    To use this library, the following attributes must be overridden in the build.gradle file to specify the custom scheme that will be used for the OAuth2 redirect {@see https://developer.android.com/guide/topics/manifest/data-element}:

    • auth_redirect_scheme

    • auth_redirect_host

    • auth_redirect_path

      • authorize

         final Result<TokenInfo> authorize(HttpClient httpClient, URL url, String username, String password, Array<String> scope)

        Asynchronously authorizes a user by making a request to the provided URL using the given credentials.

        This function sends a POST request to the authorization server with the specified username, password, and optional scope. It returns a Result containing a TokenInfo object if the authorization is successful, or an exception if an error occurs.

        Parameters:
        httpClient - The HttpClient used to make the network request.
        url - The URL of the authorization endpoint.
        username - The username used for authentication.
        password - The password used for authentication.
        scope - An optional array of scopes to request authorization for.
        Returns:

        A Result containing a TokenInfo object if authorization is successful, or an exception if the request fails.

      • refresh

         final Result<TokenInfo> refresh(HttpClient httpClient, URL url, String refreshToken, Array<String> scope)

        Refresh tokens are issued to the client by the authorization server and are used to obtain a new access token when the current access token becomes invalid or expires, or to obtain additional access tokens with identical or narrower scope.

        Because refresh tokens are typically long-lasting credentials used to request additional access tokens, the refresh token is bound to the client to which it was issued.

        Parameters:
        httpClient - The HttpClient used to make the network request.
        url - The URL for the OpenID Connect service provider issuer.
        refreshToken - The refresh token previously issued by the authorization server.
        scope - The scope of the access request.
